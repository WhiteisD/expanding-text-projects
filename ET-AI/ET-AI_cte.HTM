AI Terms to Learn
What changes on NUC (very briefly)

Build (The process of compiling source code into an executable or library, often tailored to hardware, OS, and performance options.) / runtime: (The environment and phase where compiled AI software executes, loads models, manages memory, and performs inference.) use a llama.cpp (A lightweight C/C++ inference engine for running LLaMA-family models efficiently on CPUs and GPUs, emphasizing local and offline use.) build that enables CUDA (NVIDIA’s parallel computing platform enabling GPU acceleration for AI workloads through specialized kernels and memory management.) (or cuBLAS (NVIDIA’s GPU-accelerated linear algebra library, heavily used for fast matrix multiplications in neural network inference.) ) so inference (The process of running a trained model to generate outputs, such as text or predictions, from new input data.) runs on the RTX GPU.

Model (A trained neural network containing learned parameters that encode patterns from data to perform tasks like language generation.) choice: move up from a 3B (A language model with roughly three billion parameters, relatively lightweight and suitable for CPUs or small GPUs.) CPU-friendly model size (The total number of parameters in a model, affecting memory usage, accuracy, and computational requirements.) to larger models (7B (A mid-sized model balancing quality and resource use, commonly used on consumer GPUs with modest VRAM.) , 13B (A larger, more capable model requiring substantial RAM or GPU VRAM, offering improved reasoning and coherence.) , maybe more) because you have 64 GB RAM and GPU VRAM (Dedicated high-speed memory on a GPU used to store model weights, activations, and intermediate tensors.) .

Quantization: (A technique that reduces numerical precision of model weights to lower memory usage and increase inference speed.) switch from “CPU survival quants (quant - Shorthand for a specific quantized version of a model, indicating reduced precision compared to full floating-point.) ” to GPU-optimized quants (A quantization format designed to run efficiently on GPUs, preserving throughput while minimizing VRAM usage.) (often GGUF (A modern binary file format for llama.cpp models, supporting metadata, quantization variants, and fast loading.) , either 5-bit (A GGUF model quantized to five bits per weight, trading slight quality loss for significant memory savings.) or 6-bit (A GGUF model using six-bit quantization, offering a balance between accuracy and reduced memory footprint.) , or even FP16 (Half-precision floating-point format commonly used on GPUs for fast, high-quality inference with moderate VRAM use.) if you go that route) for better quality at good speed.

Context length: (The maximum number of tokens a model can process at once before older tokens are discarded.) raise context (The sequence of tokens the model can attend to when generating output, including prompt and prior responses.) from 2K (A context window of roughly 2,000 tokens, sufficient for short conversations or documents.) to 8K or 16K (Extended context windows allowing much longer prompts, conversations, or documents at increased memory cost.) (as desired) because RAM allows it.

GPU offload setting: (A setting change you can make in ...) set how many layers (The number of transformer layers assigned to GPU execution, directly controlling VRAM usage and performance.) to offload to GPU (The practice of running part of the model’s computation on the GPU instead of the CPU.) (in llama.cpp this is the key lever (In llama.cpp, the primary performance knob: increase GPU layers until VRAM is full but not exceeded.) ) and tune (Adjusting how much of the model runs on GPU to maximize speed without exceeding available VRAM.) it to fit VRAM.

Threads (The number of parallel CPU execution threads used during inference, affecting speed and system load.) / batching: (Processing multiple tokens or requests together to improve hardware utilization and throughput.) CPU threads matter less; you’ll tune batch size (The number of tokens or sequences processed simultaneously during inference, impacting speed and memory use.) / flash attention options (Optimized attention algorithms that reduce memory bandwidth and latency, especially beneficial for long contexts.) (if enabled) for throughput.


That’s basically it: same architecture, different build + bigger model + more context + GPU offload tuning.
